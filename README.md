# PlatformForge

GitOps platform services management for Kubernetes. Ansible bootstraps Argo CD; Argo CD owns and deploys all platform services via ApplicationSets.

PlatformForge is designed to work alongside [ClusterForge](https://github.com/brianbeck/ClusterForge) (cluster provisioning) and [DevExForge](https://github.com/brianbeck/DevExForge) (developer experience).

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                     PlatformForge Repo                       │
│                                                             │
│  ┌──────────────┐  ┌────────────────┐  ┌─────────────────┐ │
│  │   Ansible     │  │ ApplicationSets │  │ Platform Values │ │
│  │              │  │ (argocd/)      │  │ (platform/)     │ │
│  │  bootstrap   │  │                │  │                 │ │
│  │  install-    │─>│ waves-stage/   │─>│ traefik/        │ │
│  │    argocd    │  │ waves-prod/    │  │ observability/  │ │
│  │  healthcheck │  │                │  │ gatekeeper/     │ │
│  │  teardown    │  │ Generated by   │  │ falco/          │ │
│  │              │  │ Ansible        │  │ trivy-operator/ │ │
│  └──────────────┘  └────────────────┘  └─────────────────┘ │
│                                                             │
│  Ansible installs        Argo CD owns all platform          │
│  Argo CD only            services via GitOps                │
└─────────────────────────────────────────────────────────────┘
```

**Single-owner model:** Ansible installs Argo CD. Argo CD deploys everything else. No dual control, no drift.

**GitOps flow:** Push changes to `platform/` values, Argo CD reconciles. Stage auto-syncs; prod requires manual sync.

## Environment Models

| | Model A | Model B |
|---|---|---|
| **Clusters** | 1 | 2 (stage + prod) |
| **Isolation** | Namespace-based | Cluster-based |
| **Argo CD** | 1 instance | 2 instances (one per cluster) |
| **Namespaces** | `falco-stage`, `falco-prod` | `falco` on each cluster |
| **When to use** | Dev/small teams | Production, compliance |

## Quick Start

### 1. Configure

```bash
cd PlatformForge/ansible
ansible-playbook playbooks/bootstrap.yml
```

Prompts for: Git repo URL, environment model (A/B), kubectl contexts, Traefik ingress, hostnames, Pi-hole DNS, secrets management (Sealed Secrets or External Secrets Operator). All answers saved and reused on re-runs.

### 2. Deploy

```bash
ansible-playbook playbooks/install-argocd.yml
```

This installs Argo CD, generates ApplicationSets, and applies them. Argo CD then deploys all platform services in sync-wave order.

### 3. Commit

```bash
cd ..
git add argocd/ && git commit -m "Generate ApplicationSets" && git push
```

### 4. Verify

```bash
cd ansible
ansible-playbook playbooks/healthcheck.yml
```

Or access UIs (if ingress enabled):
- Argo CD: `https://argocd-{stage,prod}.<domain>`
- Grafana: `https://grafana-{stage,prod}.<domain>`
- Prometheus: `https://prometheus-{stage,prod}.<domain>`

### 5. Iterate

```bash
# Edit platform values
vim platform/falco/base-values.yaml
git add -A && git commit -m "Update Falco rules" && git push
# Argo CD auto-syncs stage
```

## Sync Wave Order

Argo CD deploys services in wave order:

| Wave | Service | Why |
|---|---|---|
| -10 | Sealed Secrets or ESO | Secrets available before other services |
| 10 | Traefik | Ingress needed for external access |
| 20 | kube-prometheus-stack | CRDs needed by ServiceMonitors |
| 30 | Gatekeeper controller | Must run before templates |
| 40 | ConstraintTemplates | Creates CRDs for constraints |
| 50 | Constraints | Uses CRDs from wave 40 |
| 60 | Falco | Needs monitoring CRDs |
| 70 | Trivy Operator | Needs monitoring CRDs |

## Platform Services

### Traefik (Ingress)
- LoadBalancer via MetalLB, HTTP->HTTPS redirect
- IngressRoutes for Argo CD, Grafana, Prometheus
- Stage: 1 replica | Prod: 2 replicas

### Observability (kube-prometheus-stack)
- Prometheus, Alertmanager, Grafana, node-exporter, kube-state-metrics
- Auto-discovers all ServiceMonitors and PrometheusRules
- Stage: 7d retention | Prod: 30d retention, 2 replicas

### OPA Gatekeeper (Policy)
- 6 ConstraintTemplates: block privileged, require labels, require limits, block latest tag, block host namespace, require non-root
- Stage: `dryrun` | Prod: `deny`
- All platform namespaces excluded

### Falco (Runtime Security)
- Modern eBPF driver (kernel 5.8+)
- Custom rules: shell in container, sensitive file reads, privilege escalation, crypto mining, unexpected connections
- NetworkPolicies, PrometheusRules for alerting

### Sealed Secrets (default) / External Secrets Operator

Manages Kubernetes Secrets securely. Choose during bootstrap:

- **Sealed Secrets (default):** Encrypt secrets in Git using `kubeseal`. No external infrastructure needed. Each cluster has its own encryption key.
- **External Secrets Operator:** Sync secrets from HashiCorp Vault or cloud providers. Requires an external secret store.

### Trivy Operator (Vulnerability Scanning)
- Continuously scans running workloads for CVEs
- Stage: 12h rescan | Prod: 24h rescan
- PrometheusRules alert on critical vulnerabilities

**Security Layer Model:**

| Layer | Tool | When | Status |
|---|---|---|---|
| 1 | Trivy CLI in CI | Build time | User implements |
| 2 | Trivy Operator | Runtime | **Implemented** |
| 3 | Trivy Operator | Continuous | **Implemented** |
| 4 | Gatekeeper + External Data | Admission | **Planned** |


Deletes ApplicationSets -> Applications -> Argo CD -> webhooks -> namespaces.

## Troubleshooting

### Argo CD controller crash-looping on initial deploy
Normal during first sync of kube-prometheus-stack (100+ resources). Controller restarts and makes progress each cycle. Resolves within 10-15 minutes. Controller memory is set to 2Gi to accommodate.

### Gatekeeper webhook blocking namespace creation
Values set `validatingWebhookCheckIgnoreFailurePolicy: Ignore`. If still blocking, clean up manually:
```bash
kubectl delete validatingwebhookconfiguration -l gatekeeper.sh/system=yes
```

### Falco pods not starting
Check init container logs for network issues:
```bash
kubectl -n <falco-ns> logs <pod> -c falcoctl-artifact-install
```
NetworkPolicy must allow HTTPS egress (port 443).

### Prometheus not scraping
Verify ServiceMonitors exist and Prometheus selector is set to `nil` (auto-discover all):
```bash
kubectl get servicemonitor -A
```
