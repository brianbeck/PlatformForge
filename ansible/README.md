# Ansible

Ansible playbooks for bootstrapping and managing PlatformForge. Ansible handles Argo CD installation, ApplicationSet templating, and lifecycle operations. **Argo CD owns all platform services** -- Ansible does not install Helm charts for platform services.

> **New users:** Use the `platformforge` CLI instead of running playbooks directly.
> See the top-level [README.md](../README.md) for `platformforge init` / `deploy` / `status` / etc.
> The CLI writes configuration and then invokes these playbooks under the hood.

## Playbooks

| Playbook | Purpose | CLI equivalent |
|---|---|---|
| `bootstrap.yml` | Configure environment (interactive prompts) | `platformforge init` |
| `install-argocd.yml` | Install Argo CD, apply ApplicationSets, register DNS | `platformforge deploy` |
| `deploy-dns.yml` | Re-register hostnames with Pi-hole | `platformforge dns` |
| `healthcheck.yml` | Verify all services across all clusters | `platformforge status` |
| `teardown.yml` | Clean removal of all services | `platformforge teardown` |

### Direct Ansible usage

These playbooks still work when called directly. They read configuration from `environments.yml` (generated by `platformforge init` or `bootstrap.yml`) and secrets from `ansible/vault/secrets.yml`.

```bash
cd ansible
ansible-playbook playbooks/install-argocd.yml
ansible-playbook playbooks/healthcheck.yml
ansible-playbook playbooks/teardown.yml
```

## Ownership Model

| Ansible owns | Argo CD owns |
|---|---|
| Environment discovery | Traefik |
| Kubectl context validation | kube-prometheus-stack |
| Ingress/DNS configuration | Gatekeeper (controller + templates + constraints) |
| Argo CD installation | Falco |
| ApplicationSet generation | Trivy Operator |
| Pi-hole DNS registration | Argo Rollouts |
| Health checks | All platform config in `platform/` |
| Teardown | |

## Roles

| Role | Used by | Purpose |
|---|---|---|
| `discover_environment` | `bootstrap.yml` | Prompt for Model A or B |
| `discover_contexts` | `bootstrap.yml` | Discover and verify kubectl contexts |
| `discover_ingress` | `bootstrap.yml` | Configure Traefik, hostnames, Pi-hole |
| `discover_secrets` | `bootstrap.yml` | Choose Sealed Secrets or External Secrets |
| `argocd_install` | `install-argocd.yml` | Install Argo CD, template and apply ApplicationSets |
| `pihole_dns` | `install-argocd.yml`, `deploy-dns.yml` | Register DNS records with Pi-hole v6 |
